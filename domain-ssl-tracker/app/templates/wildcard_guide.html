🔒 Free Wildcard SSL Certificate Guide

What is a Wildcard SSL?

A wildcard SSL certificate (*.example.com) covers all subdomains under a domain with a single certificate — e.g. app.example.com, api.example.com, admin.example.com.

Let's Encrypt issues free wildcard certs, but they require a DNS-01 challenge — you prove domain ownership by adding a TXT record to your DNS.

Method 1 — Manual (Any DNS Provider)

Use this if your DNS provider has no API plugin.

Install Certbot
sudo apt install certbot
Request wildcard certificate
sudo certbot certonly \
  --manual \
  --preferred-challenges dns \
  -d "*.example.com" \
  -d "example.com"

Certbot will prompt you to add a DNS TXT record:

DNS TXT record to add
Name : _acme-challenge.example.com
Value: AbCdEfGhIjKlMnOpQrStUvWxYz123456789
  1. Log in to your DNS provider
  2. Add the TXT record shown by Certbot
  3. Wait 1–2 minutes for DNS propagation
  4. Press Enter in the terminal
  5. Cert files saved to /etc/letsencrypt/live/example.com/

Method 2 — Automated (DNS API)

Full automation — no manual steps. Requires a supported DNS provider.

☁️ Cloudflare
pip install certbot-dns-cloudflare

# Create credentials file
cat > /etc/letsencrypt/cloudflare.ini << EOF
dns_cloudflare_api_token = YOUR_TOKEN
EOF
chmod 600 /etc/letsencrypt/cloudflare.ini

sudo certbot certonly \
  --dns-cloudflare \
  --dns-cloudflare-credentials /etc/letsencrypt/cloudflare.ini \
  -d "*.example.com" -d "example.com"
🟠 AWS Route53
pip install certbot-dns-route53

# Ensure AWS credentials configured
# (~/.aws/credentials or IAM role on EC2)

sudo certbot certonly \
  --dns-route53 \
  -d "*.example.com" -d "example.com"
🔵 DigitalOcean
pip install certbot-dns-digitalocean

cat > /etc/letsencrypt/do.ini << EOF
dns_digitalocean_token = YOUR_DO_TOKEN
EOF
chmod 600 /etc/letsencrypt/do.ini

sudo certbot certonly \
  --dns-digitalocean \
  --dns-digitalocean-credentials /etc/letsencrypt/do.ini \
  -d "*.example.com" -d "example.com"

Auto-Renewal Setup

Let's Encrypt certs expire every 90 days. Set up auto-renewal once.

Test renewal (dry run)
sudo certbot renew --dry-run
Check systemd timer (Ubuntu 24)
sudo systemctl status certbot.timer
Or add a cron job manually
echo "0 3 * * * root certbot renew --quiet" | sudo tee /etc/cron.d/certbot

Use the Cert with Nginx

/etc/nginx/sites-available/your-site
server {
    listen 443 ssl;
    server_name *.example.com example.com;

    ssl_certificate     /etc/letsencrypt/live/example.com/fullchain.pem;
    ssl_certificate_key /etc/letsencrypt/live/example.com/privkey.pem;

    location / {
        proxy_pass http://127.0.0.1:8000;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
    }
}

server {
    listen 80;
    server_name *.example.com example.com;
    return 301 https://$host$request_uri;
}

Important Notes

  • Wildcard certs cover one level only — *.example.com covers app.example.com but NOT api.app.example.com
  • You need a separate entry for the apex domain (example.com) — add -d "example.com" alongside the wildcard
  • This tracker auto-detects wildcard certs via the Subject Alternative Name (SAN) field and shows them as wildcard in the dashboard
  • Full automation requires a DNS provider with an API — manual method works with any provider
